* TIL 및 최종 학습 기록 정리


학습 내용

이번 주에는 프로그래머스를 활용하여 실전 코딩테스트를 진행하고, 지금까지 학습한 알고리즘 내용을 종합적으로 복습하였다. 제한된 시간 안에서 문제를 해결하는 모의 코딩테스트를 통해 문제를 분석하고 적절한 알고리즘을 선택하는 연습을 수행하였다.

또한 자료구조(Stack, Queue), 그래프 탐색(DFS, BFS), 그리디 알고리즘, 이분탐색, 동적계획법(DP) 등 이전 주차에 학습한 내용을 다시 정리하며 다양한 유형의 문제를 해결하였다. 문제를 풀기 전 알고리즘 유형을 먼저 파악하고 해결 전략을 세우는 과정의 중요성을 다시 한번 확인할 수 있었다.

이번 활동에서는 팀원들과 각자의 풀이 과정을 발표하고 코드 리뷰를 진행하였다. 같은 문제를 해결하더라도 알고리즘 선택과 구현 방식이 서로 다를 수 있다는 점을 비교하며 더 효율적인 코드 작성 방법을 학습하였다. 마지막으로 GitHub와 TIL을 정리하며 지금까지의 학습 내용을 체계적으로 기록하였다.

어려웠던 점

모의 코딩테스트에서는 제한된 시간 안에 문제를 해결해야 했기 때문에, 알고리즘을 알고 있더라도 어떤 유형의 문제인지 빠르게 파악하는 과정이 가장 어려웠다. 또한 구현 과정에서 사소한 조건을 놓쳐 오답이 발생하는 경우가 있었으며, 시간 관리의 중요성을 체감할 수 있었다.


해결 방법

문제를 바로 구현하기보다 먼저 입력과 출력, 조건을 분석하고 사용할 알고리즘을 결정한 뒤 코드를 작성하는 습관을 들였다. 문제를 해결한 이후에는 팀원들과 풀이 방법을 비교하며 더 간결하거나 효율적인 구현 방법을 확인하고, 코드 리뷰를 통해 부족한 부분을 보완하였다.

느낀 점

이번 활동을 통해 알고리즘 개념을 이해하는 것뿐만 아니라 실제 코딩테스트 환경에서 빠르고 정확하게 문제를 해결하는 능력이 중요하다는 것을 느꼈다. 또한 GitHub를 활용하여 주차별 학습 내용을 꾸준히 기록하고 팀원들과 코드 리뷰를 진행하면서 협업 능력과 문제 해결 능력을 함께 향상시킬 수 있었다. 앞으로도 다양한 유형의 문제를 꾸준히 풀이하며 알고리즘 활용 능력을 더욱 발전시켜 나가고 싶다.
